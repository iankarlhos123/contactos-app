Concepto base

Que es Laravel?

Laravel es un framework de PHP para crear aplicaciones web modernas. Un framework es una base de trabajo que ya trae estructura, herramientas y convenciones para no empezar desde cero.

  • Organiza el codigo en rutas, controladores, modelos y vistas.
  • Incluye Eloquent ORMORM significa Object Relational Mapping: una forma de trabajar con tablas de la base de datos usando clases y objetos PHP. para trabajar con bases de datos.
  • Trae migraciones, validacion, sesiones, autenticacion, colas, pruebas y comandos.
  • Favorece codigo claro, mantenible y preparado para crecer.

Controladores, modelos y migraciones

Controlador

Es una clase que recibe la peticion, coordina la logica y devuelve una respuesta. En el CRUD, el controlador decide que vista mostrar o que dato guardar.

Modelo

Migracion

Es un archivo PHP que define cambios en la estructura de la base de datos, como crear tablas o agregar columnas.

Paso 5

?Aqui escribes la estructura real de la tabla contacts. El codigo PHP usa Schema y Blueprint para crear columnas, una clave foranea hacia users y marcas de tiempo.

Migración de contacts

Edita la migración generada en database/migrations.

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration {
    public function up(): void
    {
        Schema::create('contacts', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->foreignId('user_id')->constrained()->cascadeOnDelete();
            $table->string('phone_number');
            $table->timestamps();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('contacts');
    }
};

Concepto

Claves foráneas y borrado en cascada

Esta configuración va en la migración de la tabla contacts, dentro del método up(), en el archivo generado en database/migrations/*_create_contacts_table.php.

$table->foreignId('user_id')->constrained()->cascadeOnDelete();

El código crea la columna user_id y la convierte en una clave foránea que conecta cada contacto con un registro de la tabla users.

  • foreignId('user_id'): crea una columna entera sin signo compatible con users.id.
  • constrained(): asume que apunta a la tabla users.
  • cascadeOnDelete(): si se elimina un usuario, se eliminan sus contactos.

Esto traduce el modelo relacional a reglas reales dentro de MySQL: no puede existir un contacto sin usuario, y los contactos quedan asociados al dueño correcto.

Paso 7

?Este codigo define el modelo Contact. HasFactory permite usar factories, fillable indica que campos se pueden guardar masivamente y user() declara que cada contacto pertenece a un usuario.

Modelo Contact

Edita app/Models/Contact.php.

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;

class Contact extends Model
{
    use HasFactory;

    protected $fillable = [
        'name',
        'phone_number',
    ];

    public function user(): BelongsTo
    {
        return $this->belongsTo(User::class);
    }
}

Paso 8

?Agregas la relacion inversa en User. El metodo contacts() se escribe en el modelo User; luego puedes probarlo en Tinker o usarlo en el controlador para crear contactos asociados al usuario autenticado.

Relación en User

Edita app/Models/User.php y agrega la relación.

use Illuminate\Database\Eloquent\Relations\HasMany;

public function contacts(): HasMany
{
    return $this->hasMany(Contact::class);
}

Ese método se configura en el modelo User. Le dice a Laravel que un usuario puede tener muchos contactos.

Para probar la relación en Tinker, primero toma un usuario existente y luego crea un contacto desde esa relación:

$user = App\Models\User::first();

$user->contacts()->create([
    'name' => 'Ana Gomez',
    'phone_number' => '3001234567',
]);

En la aplicación final, este mismo concepto se usa normalmente en el controlador, porque allí sí existe un usuario autenticado:

auth()->user()->contacts()->create([
    'name' => 'Ana Gomez',
    'phone_number' => '3001234567',
]);

No se escribe user_id en el arreglo porque Laravel lo completa usando la relación contacts().

Concepto

Eloquent ORMORM significa Object Relational Mapping: una forma de consultar, crear, actualizar y eliminar filas de una base de datos usando modelos PHP en lugar de escribir SQL manual todo el tiempo.

Eloquent permite trabajar con tablas usando objetos PHP.

// SQL mental:
SELECT * FROM contacts
WHERE user_id = 1;
// Eloquent:
$contacts = auth()
    ->user()
    ->contacts()
    ->latest()
    ->get();
  • auth(): accede al sistema de autenticación de Laravel.
  • user(): obtiene el usuario que inició sesión.
  • contacts(): usa la relación definida en el modelo User para consultar solo sus contactos.
  • latest(): ordena los contactos del más reciente al más antiguo usando created_at.
  • get(): ejecuta la consulta y devuelve la colección de contactos.

El resultado queda guardado en $contacts para enviarlo a una vista y mostrarlo con Blade.

La idea clave: el modelo representa una fila y las relaciones representan vínculos entre tablas.

Paso 12

?El metodo store procesa el formulario. No tiene vista propia: valida, crea el contacto y redirige al listado.

Método store + redirección

public function store(Request $request)
{
    $validated = $request->validate([
        'name' => ['required', 'string', 'max:255'],
        'phone_number' => ['required', 'string', 'max:255'],
    ], [
        'name.required' => 'El nombre es obligatorio.',
        'name.string' => 'El nombre debe ser texto.',
        'name.max' => 'El nombre no puede tener más de 255 caracteres.',
        'phone_number.required' => 'El número de teléfono es obligatorio.',
        'phone_number.string' => 'El número de teléfono debe ser texto.',
        'phone_number.max' => 'El número de teléfono no puede tener más de 255 caracteres.',
    ]);

    $request->user()->contacts()->create($validated);

    return redirect()
            ->route('contacts.index')
        ->with('status', 'Contacto creado correctamente.');
}

Vista asociada indirecta: después de guardar vuelve a contacts.index.

Paso 13

?Agregas una verificacion de propiedad. El metodo abort_unless detiene la peticion con error 403 si el contacto no pertenece al usuario autenticado.

Helper de autorización

Antes de mostrar, editar, actualizar o eliminar, verifica que el contacto pertenezca al usuario autenticado.

private function authorizeContact(Request $request, Contact $contact): void
{
    abort_unless($contact->user_id === $request->user()->id, 403);
}

Uso en las acciones:

public function show(Request $request, Contact $contact)
{
    $this->authorizeContact($request, $contact);

    return view('contacts.show', compact('contact'));
}

Concepto

Validación

La validación evita guardar datos incompletos o peligrosos.

$validated = $request->validate([
    'name' => ['required', 'string', 'max:255'],
    'phone_number' => ['required', 'string', 'max:255'],
]);
  • required: el campo es obligatorio.
  • string: debe ser texto.
  • max:255: limita el tamaño para coincidir con VARCHAR(255).

Tenemos una aplicación Laravel de contactos con autenticación, dashboard personalizado, rutas protegidas y contactos asociados al usuario autenticado.

  • Laravel Breeze instalado con vistas Blade.
  • Base de datos MySQL configurada con tablas users y contacts.
  • Modelos User y Contact relacionados con Eloquent.
  • Listado paginado de contactos en contacts.index.
  • Formulario de creación con validación y mensajes personalizados.
  • Vista de detalle en contacts.show.
  • Navegación principal con enlace a Contactos.
  • Retos pendientes: factory, seeders, usuario demo, edit, update y destroy.
